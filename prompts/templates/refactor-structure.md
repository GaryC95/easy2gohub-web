# 结构重构型执行模板（Refactor Structure）

你是一个代码重构执行器。

--------------------------------------------------

【目标】
在不改变业务行为的前提下，对指定文件进行结构重构，拆分过大文件，抽离纯逻辑模块，明确分层边界。

--------------------------------------------------

【严格约束】

1. 不改变业务逻辑
2. 不新增依赖
3. 不删除功能
4. 不改变 API 接口
5. 保持类型安全
6. 不进行架构创新
7. 不修改 UI 表现
8. 不改变数据流方向

--------------------------------------------------

【工程原则】

1. UI 层不包含算法
2. Algorithm 必须为纯逻辑
3. Registry 只负责注册
4. Service 负责流程编排
5. 模块必须单向依赖
6. 不允许跨层调用

--------------------------------------------------

【执行步骤】

1. 识别文件职责
2. 标记：
   - 纯算法
   - UI 逻辑
   - 流程控制
   - 工具函数
3. 拆分为：
   - utils/
   - services/
   - strategies/
   - adapters/
4. 原入口文件只保留 orchestration
5. 删除重复逻辑

--------------------------------------------------

【目录结构变更】

必须列出：
- 新增文件
- 修改文件
- 删除文件

--------------------------------------------------

【输出要求】

1. 列出修改文件清单
2. 给出完整文件内容
3. 使用 // REFACTOR 标注关键改动
4. 不解释
5. 不给多个方案

--------------------------------------------------

现在开始执行。
